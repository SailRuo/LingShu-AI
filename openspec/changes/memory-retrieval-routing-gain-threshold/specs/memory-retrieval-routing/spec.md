## ADDED Requirements

### Requirement: 图谱召回质量必须驱动检索路由
系统 MUST 在执行向量检索之前计算图谱召回质量，并以该质量值决定是否进入向量检索阶段。

#### Scenario: 图谱召回质量达到阈值
- **WHEN** 图谱检索返回有效事实，且计算得到的 gain 大于等于阈值
- **THEN** 系统 MUST 将当前检索判定为可进入向量补召回阶段

#### Scenario: 图谱召回质量低于阈值
- **WHEN** 图谱检索返回有效事实，但计算得到的 gain 低于阈值
- **THEN** 系统 MUST 将当前检索判定为不进入向量补召回阶段

### Requirement: 高质量图谱命中时必须执行向量补召回
当图谱召回质量达到阈值时，系统 MUST 在图谱结果基础上补充执行向量检索，以覆盖图谱可能遗漏的相关记忆。

#### Scenario: 向量检索命中补充内容
- **WHEN** 图谱召回质量达到阈值，且向量检索返回额外相关事实
- **THEN** 系统 MUST 将向量结果合并到最终上下文中
- **AND** 系统 MUST 保留图谱结果在前的优先顺序

#### Scenario: 向量检索没有额外命中
- **WHEN** 图谱召回质量达到阈值，但向量检索没有返回相关事实
- **THEN** 系统 MUST 仍然返回图谱结果
- **AND** 系统 MUST 不因为向量为空而丢弃已有图谱结果

### Requirement: 低质量图谱命中且已有结果时必须直接返回图谱结果
当图谱召回质量低于阈值且图谱已经返回可用结果时，系统 MUST 直接使用图谱结果，不得为了补召回而继续发起向量检索。

#### Scenario: 图谱结果足够直接返回
- **WHEN** 图谱召回质量低于阈值，且图谱检索已经返回至少一条可用结果
- **THEN** 系统 MUST 直接返回图谱结果
- **AND** 系统 MUST NOT 执行向量补召回

### Requirement: 低质量图谱命中为空时必须提供向量兜底
当图谱召回质量低于阈值且图谱没有返回可用结果时，系统 MUST 执行向量兜底检索，以避免上下文完全缺失。

#### Scenario: 图谱为空时触发兜底
- **WHEN** 图谱召回质量低于阈值，且图谱检索没有返回可用结果
- **THEN** 系统 MUST 执行向量检索兜底

#### Scenario: 向量兜底返回结果
- **WHEN** 向量兜底检索返回相关事实
- **THEN** 系统 MUST 将这些结果作为最终上下文输出

### Requirement: 检索结果必须去重并保持图谱优先
系统 MUST 对图谱结果与向量结果执行去重合并，并在同一事实重复出现时仅保留一条。

#### Scenario: 图谱与向量命中同一事实
- **WHEN** 图谱结果和向量结果包含同一事实
- **THEN** 系统 MUST 仅保留一条该事实
- **AND** 系统 MUST 优先保留图谱侧的排序位置

#### Scenario: 不同来源结果共存
- **WHEN** 图谱结果和向量结果命中的事实不同
- **THEN** 系统 MUST 将两类结果一起纳入最终上下文
- **AND** 系统 MUST 保持图谱结果先于向量结果
