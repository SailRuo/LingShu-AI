## ADDED Requirements

### Requirement: 系统必须提供 G A M R A G 标准行为
系统 MUST 将记忆检索定义为 `GAM-RAG` 标准流程：实体提取、图谱激活、增益判断、路由决策与结果融合。

#### Scenario: 执行标准流程
- **WHEN** 用户发起记忆检索请求
- **THEN** 系统 MUST 先提取实体
- **AND** 系统 MUST 基于实体激活图谱事实
- **AND** 系统 MUST 根据增益值决定是否进入向量阶段
- **AND** 系统 MUST 输出融合后的最终上下文

### Requirement: 图谱激活必须基于实体命中
系统 MUST 仅使用当前用户可见的记忆事实进行图谱激活，并通过实体命中判定相关事实。

#### Scenario: 实体命中激活事实
- **WHEN** 用户消息中包含可提取实体
- **THEN** 系统 MUST 在当前用户事实中查找实体命中的事实
- **AND** 系统 MUST 将命中的事实视为激活事实

#### Scenario: 没有实体时不进入标准激活
- **WHEN** 用户消息中没有可提取实体
- **THEN** 系统 MUST 将图谱增益视为 0
- **AND** 系统 MUST NOT 将该请求判定为可进入向量补召回

### Requirement: 增益计算必须可解释
系统 MUST 基于实体覆盖率和激活事实重要度计算增益，并记录该增益用于路由判定。

#### Scenario: 计算增益
- **WHEN** 图谱激活得到一组事实
- **THEN** 系统 MUST 计算实体覆盖率
- **AND** 系统 MUST 计算激活事实的重要度均值
- **AND** 系统 MUST 输出最终增益值

### Requirement: 高增益场景必须允许向量补召回
当增益达到阈值时，系统 MUST 允许执行向量补召回，以提升召回覆盖率。

#### Scenario: 进入向量补召回
- **WHEN** 增益大于等于阈值
- **THEN** 系统 MUST 进入向量检索阶段
- **AND** 系统 MUST 将向量结果作为图谱结果的补充

### Requirement: 低增益且图谱有结果时必须直接返回图谱结果
当增益低于阈值且图谱已经返回可用结果时，系统 MUST 直接返回图谱结果，不得强制进入向量补召回。

#### Scenario: 低增益直接返回
- **WHEN** 增益低于阈值，且图谱返回至少一条可用事实
- **THEN** 系统 MUST 直接返回图谱结果
- **AND** 系统 MUST NOT 执行额外的向量补召回

### Requirement: 图谱无结果时必须提供向量兜底
当图谱未返回可用结果时，系统 MUST 提供向量兜底，以避免检索完全失败。

#### Scenario: 图谱为空时兜底
- **WHEN** 图谱激活结果为空
- **THEN** 系统 MUST 执行向量兜底检索

### Requirement: 最终结果必须保持图谱优先并去重
系统 MUST 对图谱结果与向量结果执行去重合并，并优先保留图谱结果的排序位置。

#### Scenario: 重复事实合并
- **WHEN** 图谱结果与向量结果命中同一事实
- **THEN** 系统 MUST 仅保留一条该事实
- **AND** 系统 MUST 保留图谱侧的优先顺序

#### Scenario: 不同来源结果合并
- **WHEN** 图谱结果与向量结果命中的事实不同
- **THEN** 系统 MUST 将两类结果共同纳入最终上下文

### Requirement: 检索过程必须可观测
系统 MUST 记录 `GAM-RAG` 检索过程中的关键观测信息，包括实体、增益、激活事实数量、路由结果和最终上下文来源。

#### Scenario: 记录检索事件
- **WHEN** 一次检索完成
- **THEN** 系统 MUST 记录实体提取结果
- **AND** 系统 MUST 记录增益值
- **AND** 系统 MUST 记录最终采用的是图谱、向量还是融合结果
