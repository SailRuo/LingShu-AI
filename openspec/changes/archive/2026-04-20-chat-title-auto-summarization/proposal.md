# Proposal: 聊天对话标题自动摘要

## 1. 目标 (Goals)
实现对话标题的自动生成逻辑，解决目前系统中所有新对话都显示为“新对话 N”的问题。通过 LLM 摘要技术，提升用户管理对话历史的效率和体验。

## 2. 背景 (Background)
目前的 `ChatSessionServiceImpl` 仅根据计数器生成静态标题。用户在拥有多个对话时，很难通过侧边栏的“新对话 1”、“新对话 2”来快速找回特定的讨论内容。

## 3. 用户故事 (User Stories)
- 作为一名用户，当我发送第一条消息并得到回复后，我希望侧边栏的对话标题能自动更新为与内容相关的简洁描述。
- 作为系统，我需要在不阻塞主消息流的前提下，异步、高效地完成标题摘要任务。

## 4. 核心方案 (Core Solution)
利用现有的 `TurnPostProcessingServiceImpl` 异步架构：
- 监听首轮对话结束。
- 后台触发轻量级 LLM 调用。
- 更新 `chat_sessions` 表中的 `title` 字段。

## 5. 范围 (Scope)
- **In-Scope**: 后端 API 链路修改、异步后处理逻辑增强、标题摘要 Prompt 设计、数据库更新。
- **Out-of-Scope**: 前端实时推送更新（第一阶段可以通过刷新或下次加载体现，或者复用现有 Websocket 通道）、历史旧对话的批量摘要。
