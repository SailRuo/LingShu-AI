## Context

当前记忆检索已经具备 `GAM-RAG` 的雏形：先抽实体，再做图谱激活，用 `gain` 判断是否继续向量检索，最后把结果合并成上下文。但这些规则散落在实现里，缺少统一的标准说明，也缺少一份能直接用于回归和讨论的判定依据。

这次设计的目标，是把现有行为抽象成一个稳定标准，降低后续改动带来的歧义。

## Goals / Non-Goals

**Goals:**
- 定义 `GAM-RAG` 的最小可接受标准。
- 统一图谱激活、`gain` 路由和结果合并的行为。
- 补足可观测信息，便于验证标准是否被满足。
- 让标准能够直接转化为测试用例和回归断言。

**Non-Goals:**
- 不重写实体抽取实现。
- 不重构 Neo4j 数据模型或事实写入链路。
- 不在本次变更中引入新的模型训练或检索算法。

## Decisions

1. **把 `GAM-RAG` 定义为“图谱激活 + 增益路由 + 结果融合”**
   - 选择这个定义，是因为当前实现已经天然包含这三部分，标准化成本最低，也最贴近现状。
   - 备选方案是把 `GAM-RAG` 解释成纯图遍历算法，但这与当前项目的内存图谱激活方式不一致。

2. **以 `gain` 作为路由标准的核心指标**
   - `gain` 同时反映实体覆盖和事实重要度，适合作为路由判断的统一入口。
   - 备选方案是引入新的评分器，但会导致标准和实现再次分裂。

3. **将“图谱优先”写入标准**
   - 当图谱已经给出可用结果时，标准要求优先保留图谱结果，再决定是否补向量。
   - 备选方案是纯分数统一排序，但会削弱图谱在知识稳定性上的优势。

4. **把可观测性纳入标准的一部分**
   - 标准不只要求“返回什么”，还要求记录“为什么返回这些结果”，包括实体、`gain`、激活事实数量和最终来源。
   - 备选方案是只看最终答案，但这不足以支持后续调参与回归。

## Risks / Trade-offs

- [标准过于贴近当前实现] → 好处是容易落地，风险是未来算法升级时标准需要更新。
- [阈值仍是经验值] → `gain` 阈值需要结合实际数据持续校准，否则标准会显得“有定义但不稳定”。
- [可观测性增加实现成本] → 记录更多检索事件会带来少量性能和代码复杂度开销。

## Migration Plan

1. 先把标准写成规格和任务，确保“什么是 `GAM-RAG`”有统一口径。
2. 在现有检索逻辑上补充事件记录和验收指标。
3. 用回归测试覆盖标准中的关键分支，确认实现和文档一致。
4. 后续如果要调整阈值或策略，只更新设计参数，不修改标准的核心定义。

## Open Questions

- `gain` 的阈值是否需要在标准中写死，还是保持为实现参数？
- 是否需要定义最低命中数，作为 `GAM-RAG` 生效的前置条件？
- 可观测性字段是否需要和现有 `MemoryRetrievalEvent` 完全对齐？
