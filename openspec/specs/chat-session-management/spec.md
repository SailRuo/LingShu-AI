## ADDED Requirements

### Requirement: 用户可以创建新的聊天会话
系统 MUST 提供创建新聊天会话的能力，并为新会话分配独立的会话标识。

#### Scenario: 创建空白新会话
- **WHEN** 用户在对话页点击“新建对话”
- **THEN** 系统 MUST 创建一条新的聊天会话记录
- **AND** 该会话 MUST 与已有会话相互独立

#### Scenario: 创建后可立即发送消息
- **WHEN** 用户在新建会话后发送第一条消息
- **THEN** 系统 MUST 将该消息写入新会话对应的历史中

### Requirement: 用户可以查看并切换聊天会话
系统 MUST 提供会话列表，并允许用户选择任意已有会话作为当前会话。

#### Scenario: 显示会话列表
- **WHEN** 用户打开对话页
- **THEN** 系统 MUST 显示至少一个可用会话入口

#### Scenario: 切换到已有会话
- **WHEN** 用户从列表中选择另一条会话
- **THEN** 系统 MUST 将该会话设为当前会话
- **AND** 后续历史加载与消息发送 MUST 作用于该会话

### Requirement: 会话历史必须严格隔离
系统 MUST 保证不同会话之间的消息、工具事件、附件和清空操作互不影响。

#### Scenario: 历史仅加载当前会话
- **WHEN** 用户加载聊天历史
- **THEN** 系统 MUST 只返回当前会话下的对话轮次

#### Scenario: 清空当前会话不会影响其他会话
- **WHEN** 用户清空当前会话历史
- **THEN** 系统 MUST 仅删除当前会话的数据
- **AND** 其他会话的数据 MUST 保持不变

### Requirement: 当前会话状态必须可恢复
系统 MUST 在刷新页面或重新连接后恢复用户上次使用的当前会话；如果原会话不可用，则 MUST 回退到一个确定性的可用会话。

#### Scenario: 页面刷新后恢复会话
- **WHEN** 用户刷新页面
- **THEN** 系统 MUST 恢复上次使用的当前会话

#### Scenario: 当前会话失效时回退
- **WHEN** 系统保存的当前会话已不存在
- **THEN** 系统 MUST 回退到最近可用会话或创建新的会话

### Requirement: 旧的无会话标识请求必须兼容
系统 MUST 对缺少显式会话标识的旧请求提供兼容回退，避免既有历史入口失效。

#### Scenario: 旧请求仍能读取历史
- **WHEN** 旧客户端仅携带用户标识请求聊天历史
- **THEN** 系统 MUST 解析到一个确定的可用会话

#### Scenario: 旧请求仍能发送消息
- **WHEN** 旧客户端仅携带用户标识发送消息
- **THEN** 系统 MUST 将该消息写入兼容回退到的会话中
