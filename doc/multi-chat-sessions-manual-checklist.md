# 多会话功能手工验证清单

## 前置准备

1. 启动后端服务与前端页面。
2. 进入“灵墟之境”对话页，确认左侧会话面板可见。

## 核心验证

1. 首次进入页面时，应自动选中一个可用会话。
2. 点击“新建对话”后，应新增一条会话并自动切换到新会话。
3. 在新会话中发送消息后，消息与回复应只出现在当前会话。
4. 切换回旧会话后，应看到旧会话自己的历史，而不是新会话的消息。
5. 再切回新会话时，应仍能看到刚才在新会话中的消息。

## 持久化与隔离

1. 刷新页面后，应恢复到上次使用的会话。
2. 在当前会话点击“清空”后，应只清空当前会话，不影响其他会话。
3. 切换不同会话反复发送消息时，历史加载和上滑分页都应只作用于当前会话。

## 连接链路

1. WebSocket 已连接时发送消息，应正常流式返回。
2. 断开并重连后，再次发送消息，应仍然落到当前选中的会话。
3. 使用 HTTP 历史接口刷新最新消息时，不应串到其他会话。
