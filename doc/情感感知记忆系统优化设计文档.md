## 一、优化目标

### 1.1 核心目标

1. **情感前置**：情感分析在对话前完成，结果注入 LLM Prompt
2. **情感感知的事实提取**：根据情感状态调整提取策略和置信度
3. **场景记忆增强**：存储完整的情感场景，支持情感关联检索
4. **记忆生命周期**：实现记忆巩固与衰减机制

### 1.2 预期效果

| 场景 | 优化前 | 优化后 |
|------|--------|--------|
| 用户说"压力大" | AI 正常回复 | AI 检索到之前的应对方式，主动建议 |
| 用户情绪激动 | 极端表述被存储为事实 | 标记为 volatile，等待后续确认 |
| 用户多次提及同一事实 | 重复存储 | 巩固记忆，提升权重 |
| 长期未访问的事实 | 永久保留 | 逐渐衰减，自动清理 |

### 3.2 情感预分析服务

#### 核心逻辑

```java
@Service
public class EmotionPreAnalysisService {
    
    private final DynamicMemoryModel dynamicMemoryModel;
    private final EmotionContextCache contextCache;
    private final ChatMessageRepository messageRepository;
    
    public EmotionContextResult analyzeWithContext(String userId, String message, String sessionId) {
        // 1. 获取情感上下文
        EmotionContext context = contextCache.getContext(userId);
        
        // 2. 获取最近对话历史
        List<ChatMessage> recentMessages = getRecentMessages(sessionId, 5);
        
        // 3. 调用情感分析器（含上下文）
        EmotionAnalyzer analyzer = AiServices.builder(EmotionAnalyzer.class)
            .chatModel(dynamicMemoryModel)
            .build();
        
        EmotionAnalysis emotion = analyzer.analyzeWithContext(
            message,
            formatHistory(recentMessages),
            context.getSummary()
        );
        
        // 4. 更新情感上下文
        contextCache.updateContext(userId, emotion);
        
        // 5. 返回完整结果
        return EmotionContextResult.builder()
            .primaryEmotion(emotion.getEmotion())
            .intensity(emotion.getIntensity())
            .trend(context.getEmotionTrend())
            .triggerKeywords(emotion.getTriggerKeywords())
            .suggestedResponseTone(emotion.getSuggestedTone())
            .build();
    }
}
```
