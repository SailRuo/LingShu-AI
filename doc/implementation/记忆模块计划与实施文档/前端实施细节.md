# 前端实施细节 (Frontend Implementation Details)

## 一、可视化组件

### 1.1 3D 银河系图谱 (InsightView.vue)
- **技术栈**: Three.js + CSS2DRenderer + Lucide Icons。
- **布局策略 (Node Orbit Layout)**:
  - **核心层 (Orbit 1)**: 中心为 `UserNode` 或高重要度/高活跃度事实。
  - **中间层 (Orbit 2)**: 稳定状态 (stable) 的事实节点。
  - **边缘层 (Orbit 3)**: 冷却中 (cool) 或相关性较低的节点。
- **交互功能**:
  - **时间回放**: 拖动进度条，按时间戳顺序动态展现记忆的“生成”与“连接”过程。
  - **聚类过滤**: 点击侧边栏 Topic（如“情感”、“兴趣”），图谱将高亮相关星云并淡化其他区域。
  - **右键治理**: 直接在图谱节点上右键进行“查看详情”或“移除节点”操作。

### 1.2 记忆流光监视器 (StreamPanel.vue)
- **目标**: 为开发者和高级用户提供 RAG 链路的“黑盒”透明化展示。
- **展示阶段**:
  - **阶段 1 (实体提取)**: 展示 HanLP 提取的关键词及图谱命中的基础事实。
  - **阶段 2 (语义召回)**: 展示 PGVector 召回的 Top-K 片段及其实时匹配分数。
  - **阶段 3 (上下文装配)**: 展示最终喂给 LLM 的 context 列表及是否有 Fallback 触发。

## 二、治理层实现

### 2.1 治理控制台 (GovernanceView.vue)
- **分页管理**: 采用后端分页 (Naive UI DataTable remote 模式)，支持海量记忆数据审计。
- **状态筛选**: 快速定位 `conflicted` (冲突) 或 `volatile` (不确定) 的记忆，方便人工修正。
- **冷库操作**:
  - **归档 (Archive)**: 手动将事实标记为归档，停止在图谱中渲染并降低检索权重。
  - **物理删除**: 联动清理 Neo4j 节点与 PGVector 向量。
  - **批量操作 (即将支持)**: 
    - **多选删除**: 引入 `NDataTable` 的勾选列，支持跨页选择或当前页全选。
    - **批量归档**: 一键处理高噪声或过期记忆片段。
  - **立即维护**: 提供手动触发 `runMemoryMaintenance` 后端任务的按钮。

## 三、性能与优化

1. **Three.js 资源管理**: 
   - 页面不可见时（通过 `IntersectionObserver` 监听），自动停止 `requestAnimationFrame` 渲染循环，节省 CPU。
   - 使用 CSS2DRenderer 渲染文本标签，避免 WebGL 中的中文字体纹理内存占用。
2. **响应式设计**: 
   - 针对不同分辨率调整 3D 场景比例。
   - 图谱操作面板支持侧边折叠，最大化展示区域。

## 四、未来展望

1. **图谱沉浸模式**: 支持 VR/XR 设备的基础交互（需研究 WebXR 支持）。
2. **记忆拓扑演进动效**: 节点合或分裂时的平滑动画过渡，更直观展现“认知重构”。
