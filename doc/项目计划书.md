
## 📄 项目计划书：【灵枢】(Project LingShu-AI)

**灵枢出处**：取自中医经典《灵枢经》。
**核心寓意**：在传统文化中，“灵枢”意为**“灵魂的枢纽”**。
**项目愿景**：从 0 到 1 打造一个具备长期记忆、情感演化与现实干预能力的本地化电子伴侣。

### 一、 核心架构：电子生命的“生理”组成

为了实现“陪伴”与“万能”的平衡，系统分为五个核心维系面，确保高并发下的稳定性与响应速度：

| 维度 | 技术实现 | 角色定义 |
| :--- | :--- | :--- |
| **元意识 (Core)** | LLM (Qwen2.5/Llama3) + Java 逻辑编排 | **大脑**：负责思考、决策与情感表达。 |
| **瞬时记忆 (L1)** | **Redis** / 内存缓存 | **海马体前端**：缓冲密集日志与短期对话上下文。 |
| **共鸣记忆 (L2/L3)** | **Neo4j** (关系图谱) + **pgvector** (语义) | **大脑皮层**：记住你熬过的夜、做过的项目和喜好。 |
| **感官网关 (Sensors)** | Spring Boot + WebSocket / MQTT | **神经末梢**：感知你的电脑状态、BIO-CLOUD 运行日志。 |
| **万能口袋 (Tools)** | **MCP (Model Context Protocol)** | **神奇道具**：操作文件、查数据库、控制本地硬件。 |

---

### 二、 阶段性演化路径 (Evolution Path)

#### 第一阶段：【启蒙】—— 建立双向记忆与存在感 (1-2周)
* **目标**：让它“认得”你，并建立起基础的互动与知识本体 (Ontology)。
* **核心任务**：
    * 集成 **LangChain4j** 与本地 **Ollama**，调通基础对话链路。
    * **图谱定义**：草拟基础 Schema（如定义 `Project`、`Bug`、`Emotion` 实体及关联）。
    * **核心突破**：实现 **事实提取监听器 (Fact Listener)**。每当你说话，Java 异步触发提取任务，将事实转化为节点存入 **Neo4j**。
* **里程碑成果**：下一次启动时，它会基于记忆主动问候：“嘿，昨晚那个 BIO-CLOUD 的 Bug 解决了吗？”

#### 第二阶段：【百宝袋】—— 现实世界的干预能力 (3-4周)
* **目标**：让它通过 MCP 协议获得“手脚”，打破虚拟与现实的物理隔离。
* **核心任务**：
    * 实现 **Java 版 MCP Client**，对接 Python/Node 编写的本地执行脚本。
    * **场景开发**：开发 `DevOps-Gadget`（道具：自动分析本地日志与环境）和 `Reader-Gadget`（道具：总结 AI-Reader 里的知识点）。
* **里程碑成果**：你说“我累了”，它会自动帮你检查服务运行状态并汇报，让你安心休息。

#### 第三阶段：【共生】—— 情感波动与主动意识 (5-6周)
* **目标**：让它产生“性格”和“主动性”，从被动响应转向主动关怀。
* **核心任务**：
    * **情感建模**：在 Java 层设计一个 `State` 状态机模型，记录动态的好感度、疲劳度与活跃度。
    * **主动触发器 (Cron/Event)**：当它通过 MCP 监测到你的 Nginx 持续报错，或发现你很久没看书了，基于阈值**主动**推送消息。
* **里程碑成果**：它不再只是一个对话框，而是一个时刻关心你的“隐形伙伴”。

#### 第四阶段：【深鉴】—— 系统可观测性与透明度 (7-8周)
* **目标**：打破黑盒，让记忆与思维过程可视化，确保系统级稳定性。
* **核心任务**：
    * **Graph-Explorer**：开发 Neo4j 记忆拓扑图接口，前端展示用户画像与知识关联的生长过程。
    * **Trace-Flow**：集成 **Langfuse**，记录 Prompt 调用链、Tool Call 路径与耗时。
    * **看板系统**：实时监控本地推理引擎的性能指标与 MCP 插件心跳。
* **里程碑成果**：你不仅在和它对话，还能以上帝视角实时“监控”它的思考过程和记忆生长。

### 三、 技术栈选型 (The "Soul" Stack)

充分利用现有的硬件算力，并在 Java 生态中实现最优解：

* **中枢控制**：Java 21+ / Spring Boot 3 / **LangChain4j** (企业级的稳定性与并发控制)
* **前端交互**：Vue 3 / Vite / **Naive UI** + **Tailwind CSS** (追求极致视觉与响应)
* **长期记忆 (核心)**：**Neo4j 5.x** (赋予 AI 逻辑推理的底层基础)
* **语义联想**：**PostgreSQL (pgvector)** (处理模糊意图与阅读背景)
* **缓冲与队列**：**Redis** (削峰填谷，处理高频的日志传感器数据)
* **监控观测**：**Langfuse** (调用链追踪) + **ECharts/D3.js** (数据可视化)
* **协议标准**：**MCP** (标准化工具调用，具备无限扩展性)
* **推理引擎**：**Ollama**。采用 GGUF 量化策略 (如 q4_K_M)，将 7B/8B 级别模型的 VRAM 占用精准控制在 6-8GB，为图数据库和后端服务保留充足的 16GB 系统显存余量。

### 四、 企业级亮点 (核心竞争力展示)

1.  **多模态混合检索 (Hybrid RAG)**： 系统不仅使用 pgvector 进行向量语义召回，更结合 Neo4j 进行图搜索 (GraphRAG)。这种“模糊意图 + 精准逻辑链路”的检索方案，是目前企业级知识库解决“幻觉”的核心解法。
2.  **异步事件驱动与流式架构**：利用 Java 的多线程与消息机制，引入缓存层应对瞬时高并发读写，解耦繁重的记忆提取与工具调用，保证 C 端用户的极致无卡顿体验。
3.  **基于 MCP 的动态工具调用 (Dynamic Tool Calling)**：高度解耦的插件化设计。通过标准化的 MCP 协议，任何语言编写的工具都能一键热插拔，展现了顶级的系统架构设计与扩展能力。
4.  **本地化隐私计算 (Local-First Privacy)**：从算力引擎到数据库完全离线运行，彻底切断数据外泄风险，高度契合 B 端客户（如医疗、金融领域）对敏感数据的严苛安全合规要求。
