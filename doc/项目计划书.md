
## 📄 项目计划书：【灵枢】(Project LingShu-AI)

**灵枢寓意**：
灵，象征智能、情感感知与主动交互，承载长期记忆，懂你所思、记你所好，时刻陪伴。
枢：意为智能体调度中枢，代表开放、兼容、可无限扩展的统御能力。它支持用户自由配置 MCP 工具扩展，兼容 OpenAI 标准 TTS、ASR 等协议，实现本地任务执行、外部工具接入与多智能体协同，是掌控数字世界的核心与入口。
**项目愿景**：从 0 到 1 打造一个具备长期记忆、情感演化与现实干预能力的本地化智能体。

### 一、核心架构：电子生命的"生理"组成

系统采用三层分层架构设计，确保高并发下的稳定性与响应速度：

| 维度 | 技术实现 | 角色定义 |
| :--- | :--- | :--- |
| **元意识 (Core)** | LLM (Qwen2.5/Llama3) + LangChain4j + Java 逻辑编排 | **大脑**：负责思考、决策与情感表达。 |
| **瞬时记忆 (L1)** | **Redis** / 内存缓存 + `EmotionContextCache` | **海马体前端**：缓冲密集日志与短期对话上下文，情感状态缓存。 |
| **共鸣记忆 (L2/L3)** | **Neo4j 5.x** (关系图谱) + **pgvector** (语义向量) | **大脑皮层**：存储事实节点 (`FactNode`)、用户节点 (`UserNode`)、情感片段 (`EmotionalEpisode`)，支持 GAM-RAG 混合检索。 |
| **感官网关 (Sensors)** | Spring Boot WebSocket + JavaFX 原生客户端 | **神经末梢**：感知用户输入状态、本地服务运行日志、语音交互信号。 |
| **万能口袋 (Tools)** | **MCP (Model Context Protocol)** + LangChain4j Tools | **神奇道具**：通过 MCP Client 对接 Python/Node 脚本，支持热插拔工具调用。 |

### 二、 阶段性演化路径 (Evolution Path)

#### 第一阶段：【启蒙】—— 建立双向记忆与存在感 ✅ **(已完成)**
* **目标**：让它"认得"你，并建立起基础的互动与知识本体 (Ontology)。
* **核心任务**：
    * ✅ 集成 **LangChain4j** 与本地 **Ollama**，调通基础对话链路 (`ChatService`, `ChatController`)。
    * ✅ **图谱定义**：实现基础 Schema（`UserNode`、`FactNode`、`EmotionalEpisode` 实体及关联）。
    * ✅ **核心突破**：实现 **情感感知事实提取器** (`EmotionAwareFactExtractor`)。每当你说话，Java 异步触发提取任务，将事实转化为节点存入 **Neo4j**。
    * ✅ **GAM-RAG 混合检索**：结合 Neo4j 图谱激活 (`findFactsByKeywords`) 与 pgvector 语义召回 (`EmbeddingSearchRequest`)，实现精准记忆检索。
* **里程碑成果**：下一次启动时，它会基于记忆主动问候："嘿，昨晚那个 BIO-CLOUD 的 Bug 解决了吗？"

#### 第二阶段：【百宝袋】—— 现实世界的干预能力 🟡 **(进行中)**
* **目标**：让它通过 MCP 协议获得"手脚"，打破虚拟与现实的物理隔离。
* **核心任务**：
    * ✅ 实现 **Java 版 MCP Client** (`McpServiceImpl`)，对接 Python/Node 编写的本地执行脚本。
    * ✅ 开发 **系统设置管理** (`SettingService`, `SettingController`)，支持动态配置 TTS/ASR 地址、Agent Prompt 等参数。
    * 🟡 **场景开发**：开发 `DevOps-Gadget`（道具：自动分析本地日志与环境）和 `Reader-Gadget`（道具：总结 AI-Reader 里的知识点）。
* **里程碑成果**：你说"我累了"，它会自动帮你检查服务运行状态并汇报，让你安心休息。

#### 第三阶段：【共生】—— 情感波动与主动意识 🟢 **(部分完成)**
* **目标**：让它产生"性格"和"主动性"，从被动响应转向主动关怀。
* **核心任务**：
    * ✅ **情感建模**：实现 `UserState` 状态机模型（PostgreSQL），记录动态的好感度 (`affinity`)、疲劳度、活跃度、关系阶段 (`relationshipStage`: 初识→熟悉→亲密→挚友)。
    * ✅ **情感分析器**：`EmotionAnalyzer` + `EmotionPreAnalysisService` 实时分析用户情绪，注入到 Prompt 中影响回复风格。
    * ✅ **主动触发器**：`ProactiveService` 定时任务检测用户消极情绪或长时间未互动，主动推送关怀消息。
    * 🟡 **Cron/Event 扩展**：当它通过 MCP 监测到你的 Nginx 持续报错，或发现你很久没看书了，基于阈值**主动**推送消息。
* **里程碑成果**：它不再只是一个对话框，而是一个时刻关心你的"隐形伙伴"。

#### 第四阶段：【深鉴】—— 系统可观测性与透明度 🟢 **(基础完成)**
* **目标**：打破黑盒，让记忆与思维过程可视化，确保系统级稳定性。
* **核心任务**：
    * ✅ **Graph-Explorer**：开发 Neo4j 记忆拓扑图接口 (`MemoryController.getGraphData`)，前端使用 `v-network-graph` + `Three.js` 展示用户画像与知识关联的生长过程。
    * ✅ **Trace-Flow**：`SystemLogService` 记录 Prompt 调用链、Tool Call 路径与耗时，支持 Redis 实时日志流 (`LogController`).
* **里程碑成果**：你不仅在和它对话，还能以上帝视角实时“监控”它的思考过程和记忆生长。

### 三、 技术栈选型 (The "Soul" Stack)

充分利用现有的硬件算力，并在 Java 生态中实现最优解：

* **中枢控制**：Java 21+ / Spring Boot 3.2.4 / **LangChain4j 1.12.1** (企业级的稳定性与并发控制)
* **前端交互**：**Vue 3** / Vite 6 / **Naive UI** + **Tailwind CSS 4** + **Three.js** (追求极致视觉与响应)
* **长期记忆 (核心)**：**Neo4j 5.x** (Spring Data Neo4j, 图谱存储 `UserNode`/`FactNode`/`EmotionalEpisode`)
* **语义联想**：**PostgreSQL 16** (**pgvector**, 向量嵌入存储与相似度检索)
* **缓冲与队列**：**Redis 7** (削峰填谷，处理高频的日志传感器数据、情感状态缓存)
* **监控观测**：**SystemLogService** (全链路日志追踪) + **v-network-graph/D3.js** (数据可视化)
* **协议标准**：**MCP** (LangChain4j MCP Client, 支持 stdio/http 传输)
* **推理引擎**：**Ollama** (GGUF 量化策略 q4_K_M, 控制 VRAM 6-8GB)
* **语音交互**：WebSocket TTS 流式播放 + HTTP ASR 识别 (JavaFX 原生客户端集成)

### 四、企业级亮点 (核心竞争力展示)

1.  **多模态混合检索 (GAM-RAG Hybrid Search)**：系统不仅使用 pgvector 进行向量语义召回，更结合 Neo4j 进行图搜索 (`findFactsByKeywords` + `EmbeddingSearchRequest`)。通过实体关键词激活图谱路径，计算增益阈值 (`GAIN_THRESHOLD=0.65`) 决定是否触发语义检索，这种"模糊意图 + 精准逻辑链路"的检索方案，是目前企业级知识库解决"幻觉"的核心解法。
2.  **情感感知记忆系统 (Emotion-Aware Memory)**：`EmotionAwareFactExtractor` 根据情绪强度 (`intensity > 0.3`) 动态调整事实提取策略，支持情感门控、事实修正删除、元数据标记 (`emotionalTone`, `triggerKeywords`)。`EmotionContextCache` 缓存最近 10 轮情感快照，提供趋势分析与回应风格建议。
3.  **异步事件驱动与流式架构**：利用 Java 的多线程与消息机制 (`ApplicationEventPublisher`, `ProactiveGreetingEvent`)，引入 Redis 缓存层应对瞬时高并发读写 (`StringRedisTemplate`)，解耦繁重的记忆提取与工具调用。流式响应采用 Reactor 模式 (`Flux<String>`)，保证 C 端用户的极致无卡顿体验。
4.  **基于 MCP 的动态工具调用 (Dynamic Tool Calling)**：高度解耦的插件化设计。`McpServiceImpl` 使用 `ConcurrentHashMap` 缓存 MCP Clients，支持运行时添加/删除/更新配置，任何语言编写的工具都能通过标准化协议一键热插拔，展现了顶级的系统架构设计与扩展能力。
5.  **本地化隐私计算 (Local-First Privacy)**：从算力引擎 (Ollama) 到数据库 (Neo4j/PostgreSQL/Redis) 完全离线运行，彻底切断数据外泄风险。JavaFX 原生客户端 (`fx-frontend`) 提供系统托盘集成、ASR/TTS 本地化配置，高度契合 B 端客户（如医疗、金融领域）对敏感数据的严苛安全合规要求。
6.  **关系演化系统 (Relationship Evolution)**：`UserState` 实体记录好感度 (`affinity: 0-100`)、关系阶段 (`初识→熟悉→亲密→挚友`)、互动次数、最后情绪。`AffinityService` 根据情感正负向 (`increaseAffinity`/`decreaseAffinity`) 动态调整关系参数，并注入到 Prompt 中影响 AI 的回应风格 (`getStyleGuide`)。

### 五、核心功能模块详解

#### 5.1 对话服务层 (`lingshu-web/controller`)

| Controller | 路由前缀 | 核心功能 | 关键 Service |
| :--- | :--- | :--- | :--- |
| `ChatController` | `/api/chat` | 文本/流式对话、历史消息管理、主动问候触发 | `ChatService`, `ProactiveService` |
| `MemoryController` | `/api/memory` | 记忆图谱可视化数据、事实节点管理、记忆维护 | `MemoryService`, `FactRepository` |
| `AgentConfigController` | `/api/agents` | Agent 人格配置、Prompt 模板管理、默认 Agent 切换 | `AgentConfigService` |
| `McpController` | `/api/mcp` | MCP 服务器配置 CRUD、工具调用状态监控 | `McpService` |
| `SettingController` | `/api/settings` | 系统全局设置 (TTS/ASR 地址、问候冷却时间) | `SettingService` |
| `SystemStatusController` | `/api/system` | 服务健康检查 (Neo4j/Ollama/MCP)、性能指标监控 | `SettingService`, `RestTemplate` |
| `LogController` | `/api/logs` | 系统日志 SSE 实时推送、日志分页查询、清理 | `SystemLogService`, `RedisTemplate` |
| `TtsController` | `/api/tts` | TTS 服务代理转发、语音合成请求处理 | `RestTemplate` |

#### 5.2 核心业务层 (`lingshu-core/service`)

| Service | 职责描述 | 关键技术点 |
| :--- | :--- | :--- |
| `ChatService` | 对话流程编排、Prompt 构建、ToolCall 执行、流式响应 | LangChain4j `AiServices`, Reactor `Flux`, ToolEventListener |
| `MemoryService` | 事实提取、GAM-RAG 检索、图谱节点持久化、向量嵌入存储 | Neo4j Cypher, pgvector, `EmbeddingModel`, 去重算法 |
| `EmotionAwareFactExtractor` | 情感感知事实提取、情绪门控、事实修正建议 | LLM 结构化输出 (`ExtractionResult`) |
| `AffinityService` | 用户关系状态管理、好感度计算、情感更新、风格指南生成 | PostgreSQL `UserState`, 关系阶段自动升级 |
| `ProactiveService` | 定时问候任务、消极情绪检测、主动关怀消息生成 | Spring `@Scheduled`, SSE 流式推送 |
| `McpService` | MCP Client 生命周期管理、工具调用代理、配置持久化 | LangChain4j `McpClient`, `StdioMcpTransport`, `StreamableHttpMcpTransport` |
| `EmotionPreAnalysisService` | 对话前情感分析、情感上下文缓存、Prompt 注入 | `EmotionContextCache`, 最近 10 轮快照追踪 |
| `EmotionalEpisodeService` | 情感片段提取、情景记忆存储、重要性评分 | Neo4j `EmotionalEpisode` 节点，触发词提取 |
| `SystemLogService` | 全链路日志追踪 (LLM/DB/Embedding)、Redis 实时推送、性能统计 | `RedisTemplate`, SSE, 计时器管理 |
| `TurnPostProcessDecider` | 回合后处理决策 (是否分析情感/提取事实/记录互动) | LLM 决策模型，规则引擎 |
