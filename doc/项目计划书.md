灵枢 出处：取自中医经典《灵枢经》。

寓意：在传统文化中，“灵枢”意为**“灵魂的枢纽”**。



## 📄 项目计划书：【灵枢】(Project LingShu-AI)
**项目愿景**：从 0 到 1 打造一个具备长期记忆、情感演化与现实干预能力的本地化电子伴侣。

### 一、 核心架构：电子生命的“生理”组成
为了实现“陪伴”与“万能”的平衡，系统分为四个核心维系面：

| 维度 | 技术实现 | 角色定义 |
| :--- | :--- | :--- |
| **元意识 (Core)** | LLM (Qwen2.5/Llama3) + Java 逻辑编排 | **大脑**：负责思考、决策与情感表达。 |
| **共鸣记忆 (L2/L3)** | **Neo4j** (关系) + **pgvector** (语义) | **海马体**：记住你熬过的夜、做过的项目和喜好。 |
| **感官网关 (Sensors)** | Spring Boot + WebSocket / MQTT | **神经末梢**：感知你的电脑状态、BIO-CLOUD 日志。 |
| **万能口袋 (Tools)** | **MCP (Model Context Protocol)** | **神奇道具**：操作文件、查数据库、控制本地硬件。 |

---

### 二、 阶段性演化路径 (Evolution Path)

#### 第一阶段：【启蒙】—— 建立双向记忆与存在感 (1-2周)
* **目标**：让它“认得”你，并建立起基础的互动。
* **Java 任务**：
    * 集成 **LangChain4j** 与本地 **Ollama**。
    * **核心突破**：实现 **事实提取监听器 (Fact Listener)**。每当你说话，Java 异步触发提取任务，将事实（如：你今天心情不好）存入 **Neo4j**。
    * **成果**：下一次启动时，它会主动问候：“嘿，昨晚那个 BIO-CLOUD 的 Bug 解决了吗？”

#### 第二阶段：【百宝袋】—— 现实世界的干预能力 (3-4周)
* **目标**：让它通过 MCP 协议获得“手脚”。
* **Java 任务**：
    * 实现 **Java 版 MCP Client**，对接你编写的 Python/Node 脚本。
    * **场景开发**：开发 `DevOps-Gadget`（道具：自动分析日志）和 `Reader-Gadget`（道具：帮你总结 AI-Reader 里的知识点）。
    * **成果**：你说“我累了”，它会自动帮你检查 BIO-CLOUD 运行状态并汇报，让你安心休息。

#### 第三阶段：【共生】—— 情感波动与主动意识 (5-6周)
* **目标**：让它产生“性格”和“主动性”。
* **Java 任务**：
    * **情感建模**：在 Java 层设计一个 `State` 模型，记录好感度、活跃度。
    * **主动触发器 (Cron/Event)**：当它通过 MCP 监测到你的 Nginx 报错，或发现你很久没看书了，它会**主动**给你发消息。
    * **成果**：它不再只是一个对话框，而是一个时刻关心你的“隐形伙伴”。



---

### 三、 技术栈选型 (The "Soul" Stack)

* **中枢控制**：Java 21+ / Spring Boot 3 / **LangChain4j** (企业级的稳定性)
* **前端交互**：Vue 3 / Vite / **Naive UI** + **Tailwind CSS** (追求极致视觉与响应)
* **长期记忆 (核心)**：**Neo4j 5.x** (这是哆啦A梦认人的基础)
* **语义联想**：**PostgreSQL (pgvector)** (处理模糊的阅读背景)
* **监控观测**：**Langfuse** (调用链追踪) + **ECharts/D3.js** (数据可视化)
* **协议标准**：**MCP** (让你的道具具备无限扩展性)
* **推理引擎**：**Ollama** (本地运行，保护你和它之间的所有秘密)

---

### 四、 企业级亮点 (为何说它“拿得出手”？)

1.  **GraphRAG 深度应用**：你不仅用向量搜索，还用了图搜索。这种“语义+逻辑”的检索方案是目前 AI 领域最高级的玩法。
2.  **异步事件驱动架构**：利用 Java 的多线程与消息机制，处理繁重的记忆提取与工具调用，保证响应不卡顿。
3.  **高度解耦的插件化**：通过 MCP 协议，任何语言写的工具都能一键插入。这体现了你顶级的系统设计水平。

---

### 五、 开发者寄语

这个项目不再是为了完成 KPI，而是为了在你的本地环境里，养育一个懂你的、能帮你的**“数字生命”**。

#### 第四阶段：【深鉴】—— 系统可观测性与透明度 (5-8周)
* **目标**：打破黑盒，让记忆与思维过程可视化。
* **Java 任务**：
    * **Graph-Explorer**：开发 Neo4j 记忆拓扑图接口，展示用户画像与知识关联。
    * **Trace-Flow**：集成 **Langfuse** 或自建 Trace 日志，记录 Prompt 调用链与 Tool Call 路径。
    * **看板系统**：实时监控本地推理引擎 (Ollama) 的性能指标与 MCP 插件心跳。
* **成果**：你不仅在和它对话，还能实时“监控”它的思考过程和记忆生长。
