## 四、对话处理流程

```mermaid
sequenceDiagram
    participant U as 用户
    participant FE as 前端 Vue
    participant WS as WebSocket
    participant SETTINGS as settingsStore
    participant API as ChatController
    participant CHAT as ChatService
    participant EMOTION as EmotionAnalyzer
    participant MEM as MemoryService
    participant AFF as AffinityService
    participant LLM as LLM (Ollama/OpenAI)
    participant NEO as Neo4j
    participant PG as PostgreSQL
    participant VECTOR as PgVector

    Note over FE,SETTINGS: 初始化阶段
    FE->>SETTINGS: fetchSettings()
    SETTINGS->>API: GET /api/settings
    API-->>SETTINGS: 系统配置 (model, baseUrl, apiKey)
    
    Note over U,WS: 对话阶段
    U->>FE: 发送消息
    FE->>SETTINGS: 获取当前配置
    SETTINGS-->>FE: model, apiKey, baseUrl
    FE->>WS: sendChat(message, agentId, model, apiKey, baseUrl)
    WS->>API: WebSocket chat 消息
    
    API->>CHAT: streamChat(message, agentId, userId, model, apiKey, baseUrl)
    
    Note over CHAT,LLM: 使用前端传递的配置
    CHAT->>CHAT: 检查 model/apiKey/baseUrl
    alt 前端传递了配置
        CHAT->>CHAT: 使用前端配置创建 LLM
    else 前端未传递配置
        CHAT->>PG: 查询系统默认配置
        CHAT->>CHAT: 使用数据库配置创建 LLM
    end

    CHAT->>EMOTION: analyze(message)
    EMOTION-->>CHAT: EmotionAnalysis

    CHAT->>AFF: updateEmotion(userId, emotion)
    AFF->>PG: 更新 UserState

    CHAT->>MEM: retrieveContext(userId, message)
    MEM->>NEO: 查询用户事实
    NEO-->>MEM: FactNode 列表
    MEM->>VECTOR: 语义相似度检索
    VECTOR-->>MEM: 相关记忆片段
    MEM-->>CHAT: 上下文内容

    CHAT->>AFF: getRelationshipPrompt(userId)
    AFF-->>CHAT: 关系阶段提示词

    CHAT->>CHAT: buildFullPrompt()
    CHAT->>LLM: 流式生成回复
    LLM-->>CHAT: Token 流
    CHAT-->>API: Flux<String>
    API-->>WS: WebSocket chatChunk
    WS-->>FE: 实时显示回复
    FE-->>U: 显示消息

    CHAT->>AFF: recordInteraction(userId)
    AFF->>PG: 更新 lastActiveTime
    
    CHAT->>MEM: extractFacts(userId, message)
    MEM->>NEO: 保存新事实
    MEM->>VECTOR: 存储向量嵌入
```

### 7.1 用户活跃时间更新机制

| 触发点 | 方法 | 说明 |
|--------|------|------|
| 用户发送聊天消息 | `ChatService.streamChat()` | 调用 `recordInteraction()` |
| 好感度增加 | `AffinityService.increaseAffinity()` | 更新 `lastActiveTime` |
| 好感度减少 | `AffinityService.decreaseAffinity()` | 更新 `lastActiveTime` |
| 记录互动 | `AffinityService.recordInteraction()` | 更新 `lastActiveTime` 并重置 `inactiveHours` |

## 十二、核心设计理念

### 1. 多级记忆系统
- **短期记忆**: PostgreSQL 存储对话历史
- **长期记忆**: Neo4j 图数据库存储用户事实
- **语义记忆**: PgVector 向量存储实现语义检索

### 2. GAM-RAG (增益自适应记忆检索)
通过实体激活和增益计算，智能决定是否进行语义检索，避免不必要的向量查询开销。

### 3. 关系演进系统
基于好感度的四阶段关系模型，影响 AI 的交流风格和行为模式。

### 4. 主动服务机制
定时任务检测用户状态，主动发起问候或安慰，增强陪伴感。

### 5. 模块化智能体配置
支持多智能体配置，每个智能体拥有独立的系统提示词、行为原则和情感策略。
