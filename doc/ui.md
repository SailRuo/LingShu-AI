设计风格总纲 (Style Guide)
关键词：赛博禅意 (Cyber-Zen)、极简主义、玻璃拟态 (Glassmorphism)、动态粒子、深色模式、呼吸感、智慧感。
配色方案：深邃墨色 (#0D0D0D) 为底，点缀灵动青色 (#00F2FF) 或苍蓝色 (#4facfe)，辅以柔和的琥珀色 (#FFB347) 表示情感状态。
字体：现代非衬线体 (Inter/Outfit)，等宽字体用于展示 AI 思考链。
1. 核心交互界面：【共鸣】 (Main Chat & Resonance)
设计目标：超越传统对话框，创造一种“伴侣”在侧的陪伴感。

Stitch Prompt:

Create a futuristic AI companion chat interface named "LingShu-AI". The design should feel premium and intimate.

Main Area: A clean, center-aligned chat flow with elegant message bubbles. Use subtle glassmorphism for the background.
AI Avatar: Instead of a static image, a dynamic, glowing "Soul Core" orb that pulses with a "breathing" animation based on AI status.
Sidebar: A "Long-term Memory" snippet showing recent facts the AI learned about the user (e.g., "Current project: BIO-CLOUD", "Mood: A bit tired").
Input Bar: A floating, minimal input field with glowing borders. Include 3 small, sleek action icons for "Pocket Tools (MCP)", "Voice", and "Memory Focus".
Aesthetics: Cyber-Zen style, dark theme, deep blue and teal gradients, high-end typography (Inter), responsive layout.

2. 记忆拓扑图：【深鉴】 (Graph-Explorer)
设计目标：可视化 Neo4j 中的复杂关系，让用户看到 AI 的“思维森林”。

Stitch Prompt:

Design a complex, interactive "Memory Topology Map" for a GraphRAG system.

Core Feature: A large, 3D-looking node-link diagram visualising relationships between user concepts (e.g., User -> Likes -> Coffee, User -> Working on -> LingShu).
Visual Style: Nodes should be glowing points of light; connections are faint, shimmering threads. Differentiate node types by subtle color coding (Facts, Emotions, Projects).
Details Panel: A side panel that appears when a node is clicked, showing the RAW data from Neo4j and the semantic connection strength.
Interface: A HUD-like overlay (Heads-Up Display). Include zoom/pan controls and a search bar for "Memory Keywords".
Vibe: Scientific, high-tech, like the neural network visualization in "Interstellar".

3. 系统看板与 MCP 工具：【万能口袋】 (MCP & Dashboard)
设计目标：展示本地性能指标与高度可扩展的插件系统。

Stitch Prompt:

Create a high-performance system dashboard for "LingShu-AI".

Performance Section: Real-time monitoring of local LLM (Ollama) inference speed, token count, and memory usage using sleek ECharts-style line graphs.
MCP Tool Grid: A modular grid showing active "Gadgets" (e.g., DevOps-Gadget, Reader-Gadget). Each card shows service status (Heartbeat), last action, and a toggle.
Trace-Flow: A vertical timeline or Sankey diagram showing the "Chain of Thought" (Prompt -> Tool Call -> Context Retrieval -> Final Answer).
Layout: Clean grid layout with a dashboard feel. Use a monochrome palette with sharp contrast for high readability.

4. 情感与状态页面：【形态演化】 (Emotional State & Settings)
设计目标：量化 AI 的性格与好感度，增加互动趣味。

Stitch Prompt:

Design a "Persona & Emotion" configuration page for an AI lifeform.

Visual Representation: A vertical bar or radar chart showing current emotional parameters: "Affinity (好感度)", "Activity (活跃度)", "Curiosity".
Mood History: A heatmap showing the evolution of the AI's mood over the week.
Settings: Minimalist toggles for "Proactive Intervention" (allowing the AI to message first if a system error is detected).
Aesthetics: Use soft, organic gradients (warm oranges to cool purples) to contrast with the technical parts of the app. Premium UI/UX with smooth micro-interactions.
使用建议：
分步生成：建议先把 【共鸣】 (Prompt 1) 丢给 Stitch，确定整体基调。
融合细节：在生成过程中，可以告诉 Stitch：“在左上角加上‘灵枢·中医经典’的中文字体 Logo 装饰，增加文化底蕴。”
强调 Native UI：既然您计划书里提到了 Naive UI，可以让 Stitch 优先模拟它的设计规范（圆角、纯净背景、极高的留白艺术）。
