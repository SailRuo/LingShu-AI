### 1.2 架构设计理念

本系统采用 **电子生命隐喻架构**，将 AI 智能体视为一个具有生理系统的数字生命体：

| 隐喻维度        | 技术实现                                                    | 生物学对应       |
| :---------- | :------------------------------------------------------ | :---------- |
| **大脑皮层**    | LangChain4j + OpenAI API 兼容 LLM(Ollama/LM Studio/GPT 等) | 思考、决策、语言生成  |
| **海马体**     | Redis + EmotionContextCache                             | 短期记忆、情感状态缓存 |
| **大脑皮层记忆区** | Neo4j + pgvector                                        | 长期记忆存储与检索   |
| **神经系统**    | WebSocket + SSE                                         | 感知输入、实时推送   |
| **运动系统**    | MCP Client + Tools                                      | 执行外部任务、工具调用 |
| **内分泌系统**   | AffinityService + EmotionService                        | 好感度、情感状态调节  |

### 3.2 技术选型理由

| **后端框架**   | Spring Boot 3.2.4                              | 企业级稳定性、依赖注入、AOP 支持                           |

| **前端框架**   | Vue 3 + Composition API                        | 响应式编程、组件复用、生态丰富                              |
| **UI 组件库** | Naive UI                                       | TypeScript 友好、主题定制、性能优秀                      |
| **3D 可视化** | Three.js + v-network-graph                     | WebGL 渲染、力导向布局、银河系效果                         |
| **LLM 引擎** | OpenAI API 兼容服务(Ollama/LM Studio/GPT-4/Claude) | 灵活部署、支持本地/云端、GGUF 量化、统一接口                    |

### 5.3 核心接口示例

#### 5.3.1 流式对话接口

```java
// Controller 层
@PostMapping(value = "/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
public Flux<String> streamChat(
    @RequestBody ChatRequest request,
    @RequestHeader(value = "X-User-Id", defaultValue = "User") String userId
) {
    return chatService.streamChat(
        request.getMessage(),
        request.getAgentId(),
        userId,
        request.getModel(),
        request.getApiKey(),
        request.getBaseUrl()
    );
}

// Service 层
public Flux<String> streamChat(String message, Long agentId, String userId, ...) {
    // 1. 情感预分析
    EmotionContextResult emotionResult = emotionPreAnalysisService.analyzeBeforeResponse(...);
    
    // 2. 记忆检索
    String longTermContext = memoryService.retrieveContext(userId, message);
    
    // 3. Prompt 构建
    String systemPrompt = promptBuilderService.buildMergedSystemPrompt(...);
    
    // 4. 流式调用 LLM
    return statelessStreamingAssistant.chatFlux(systemPrompt, userPrompt, ...);
}
```

## 6. 核心流程设计

### 6.1 对话处理全流程

```mermaid
flowchart TD
    Start([收到用户消息]) --> PreAnalysis[情感预分析]
    PreAnalysis --> GetEmotion[从 Redis 读取情感状态]
    GetEmotion --> InjectEmotion[情感 Prompt 注入]
    
    InjectEmotion --> MemoryRetrieval[记忆检索 GAM-RAG]
    MemoryRetrieval --> GraphActivate[Neo4j 图谱激活]
    GraphActivate --> CalcGain{增益 >= 0.65?}
    CalcGain -->|Yes| VectorSearch[pgvector 语义检索]
    CalcGain -->|No| SkipVector[跳过语义检索]
    VectorSearch --> RankFacts[事实排序]
    SkipVector --> RankFacts
    
    RankFacts --> BuildPrompt[构建 System Prompt]
    BuildPrompt --> CallLLM[调用 LLM 服务 OpenAI API 兼容]
    CallLLM --> StreamResponse[流式返回给用户]
    
    StreamResponse --> PostProcess[回合后处理]
    PostProcess --> DecideAction[LLM 决策是否处理]
    DecideAction --> AnalyzeEmotion{分析情感？}
    AnalyzeEmotion -->|Yes| UpdateEmotion[更新 UserState]
    AnalyzeEmotion -->|No| ExtractFacts{提取事实？}
    
    ExtractFacts -->|Yes| EmotionAwareExtract[情感感知事实提取]
    ExtractFacts -->|No| RecordInteraction{记录互动？}
    
    EmotionAwareExtract --> SaveToNeo4j[保存事实到 Neo4j]
    SaveToNeo4j --> StoreEmbedding[存储向量到 pgvector]
    StoreEmbedding --> RecordInteraction
    
    RecordInteraction -->|Yes| UpdateAffinity[更新好感度]
    RecordInteraction -->|No| End([流程结束])
    UpdateAffinity --> End
```

### 6.2 情感感知事实提取流程

```mermaid
sequenceDiagram
    participant SM as ChatService
    participant MP as TurnPostProcessDecider
    participant EF as EmotionAwareFactExtractor
    participant LLM as Ollama
    participant MS as MemoryServiceImpl
    participant FR as FactRepository
    participant ES as EmbeddingStore
    
    SM->>MP: 调用 postProcessTurn()
    MP->>MP: LLM 决策 isExtractFacts=true
    
    MP->>EF: analyzeWithEmotion() message, facts, emotion
```

### 9.6 容错与降级策略

```mermaid
flowchart TD
    Start[请求进入] --> Validate{参数校验}
    Validate -->|失败| ReturnError[返回 400 Bad Request]
    Validate -->|成功| TryDB{数据库可用？}
    
    TryDB -->|否 | FallbackCache[降级到 Redis 缓存]

    Retry -->|Yes| Wait[等待 100ms]
    Wait --> ExecuteDB
    Retry -->|No| CircuitBreaker[熔断器打开]
```
