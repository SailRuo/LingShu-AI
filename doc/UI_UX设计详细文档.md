# 📄 【灵枢】(Project LingShu-AI) 前端 UI/UX 详细设计文档

## 1. 概述 (Overview)

### 1.1 项目愿景
【灵枢】旨在从 0 到 1 打造一个具备长期记忆、情感演化与现实干预能力的本地化电子伴侣。前端界面作为用户与“数字生命”交互的唯一物理窗口，必须完美承载“生命感”（陪伴与呼吸）与“专业感”（全栈监控与极客控制）的双重职责。

### 1.2 设计目标
* **极致视觉 (Premium Aesthetics)**：打破传统管理后台 UI 的刻板印象，追求赛博朋克与东方禅意结合的视觉冲击力。
* **白盒化 (Observability)**：将复杂的 AI 思考链路、大模型推理耗时与记忆生长过程，通过可视化手段毫无保留地展示。
* **沉浸共生 (Immersion & Co-existence)**：通过动效、环境色渲染与 HUD (抬头显示) 元素，让用户感知到一个“活在本地”的数字灵魂。

---

## 2. 视觉风格与调性 (Aesthetics & Style)

### 2.1 风格定义：Cyber-Zen (赛博禅)
* **赛博 (Cyber)**：硬核的底层算力体现。包含 HUD 数据面板、发光边框、终端代码块、扫描线与点阵背景。
* **禅意 (Zen)**：中式名字的克制与呼吸感。包含空灵的背板色、磨砂玻璃 (Glassmorphism) 的通透感、流畅的贝塞尔曲线与有节奏的呼吸光效。

### 2.2 核心视觉元素
* **玻璃拟态 (Glassmorphism)**：使用高模糊度 (`backdrop-blur-xl`) 的毛玻璃作为各模块的悬浮背景板。
* **动效氛围 (Atmosphere)**：包含网格背景的极缓慢平移、环境光偏移 (Mesh Gradient)、关键动作时的扫描线叠加。
* **HUD 装饰边界**：在屏幕角落、容器边缘添加伪代码（如 `SYS.MEM.ALLOC_`）、系统坐标、实时运行参数等装饰性等宽文本，营造“控制舱”沉浸感。

* **核心标题 / HUD 数值**: `Outfit` 或 `Fira Code` (强调极客感与数字可读性)。

* **代码与日志**: `JetBrains Mono` 或 `Monaco` (严格等宽，用于展示 Nginx 报错、JSON Payload 及终端指令)。

### 3.3 阴影与光效 (Glow & Shadow)
* **文本辉光 (Text Glow)**: `drop-shadow(0 0 8px var(--color-primary))`
* **容器发光 (Box Glow)**: `box-shadow: 0 0 15px var(--color-outline)`

## 4. 布局系统 (Layout System)

采用 **三栏式 HUD 响应式布局 + 全局隐藏层**，确保高密度信息的井然有序：

1.  **Header (顶栏)**: 站点标识、神经连接状态指示灯、主题切换器、全屏沉浸模式 Toggle。
2.  **Left Sidebar (导航侧边栏)**: 核心能力舱 (意念共鸣, 深度洞察, 基础设施) 以及 MCP 插件挂载状态指示。
3.  **Main Area (主显示区)**:
    * **动态主视口**: 会根据当前所选模块，无缝切换为对话流或全屏拓扑画布。
    * **底部控制台 (Terminal Input)**: 浮动在底部的沉浸式对话/命令混合输入框。
4.  **Right Panel (详情观测栏)**: 实时剖析被选中的记忆节点，显示详细 Payload、认知权重条、精确到毫秒的时间戳。
5.  **Footer (系统状态栏)**: 常驻底部的极细状态条，展示 Ollama 显存余量、Neo4j 延迟与灵枢的“心跳频率”。
6.  **【新增】Global Command Layer (全局指令层)**: 隐藏的超级面板，通过 `Ctrl + K` (或 `Cmd + K`) 随时唤醒。

## 5. 核心交互组件 (Core Components)

### 5.1 记忆拓扑网络 (Graph-Explorer)
* **节点设计**: 彻底摒弃实心圆。采用空心圆环、虚线描边与中心发光呼吸灯。
* **边缘连线**: 使用带有张力的贝塞尔曲线。当数据被检索时，连线上有代表数据包的“光点”高速流过。
* **聚焦交互**: 鼠标悬停节点放大，触发光晕增强；点击后，全局视角平滑飞跃至该节点，右侧栏同步解析记忆数据。

### 5.2 思考流可视化 (Chain of Thought Trace)
* **形态**: 对话气泡上方弹出的微型折叠面板，类似终端命令行。
* **作用**: 在大模型响应前，实时打印后端动作，打破黑盒。
* **示例表现**: `[sys] 正在唤醒海马体... -> [neo4j] 匹配到历史节点 {BIO-CLOUD Bug} -> [mcp] 调用本地终端读取 Nginx 状态 -> [llm] 开始生成回复`

### 5.3 全局极客指令盘 (Command Palette)
* **形态**: 屏幕中央弹出的 Spotlight 风格磨砂黑框。
* **交互**: 支持自然语言或以 `/` 开头的极客指令（如 `/mcp restart nginx`，`/mem flush today`）。
* **意义**: 赋予资深开发者最硬核、最快捷的本地系统控制权。

### 5.4 认知权重刻度尺 (Cognitive Weight Ruler)
* **形式**: 断点式的分段刻度进度条，带有金属质感的 Shimmer (流光) 扫过效果。
* **意图**: 量化展示灵枢对当前用户情绪或某项知识点的“确信度”与“掌握深度”。

## 6. 重点页面设计 (Key Views)

### 6.1 意念共鸣 (Resonance Console)
* **核心功能**: 日常对话沟通与指令下达的主阵地。
* **UX 特色**: 弱化传统气泡的边界感，采用连续流式的排版。背景始终以极低的透明度，缓慢渲染近期对话激活的记忆拓扑碎片。

### 6.2 深度洞察 (Matrix Insight)
* **核心功能**: 全量 Neo4j 记忆图谱的上帝视角探索。
* **UX 特色**: 支持无级缩放与画布漫游。提供“修剪 (Pruning)”工具，允许用户手动抹除或合并冗余的 AI 记忆节点。背景采用深空点阵，营造浩瀚感。

### 6.3 基础设施监控 (Infra Dashboard)
* **核心功能**: 直观展现本地算力引擎 (Ollama) 与数据库的健康度。
* **UX 特色**: 大量运用仪表盘、ECharts 实时心电图折线。精准显示 RTX 4080 Super 的 VRAM 占用（如：`8.4GB / 16GB`），以及 MCP 探针的实时吞吐量。

## 7. 动效与情绪设计 (Animation & Emotion-Driven UI)

除了基础流畅性，UI 的动效必须直接反映“灵枢”当前的内部状态机。

### 7.1 基础系统动效
| 场景 | 动画方案 | 效果定义 |
| :--- | :--- | :--- |
| **系统唤醒/页面入场** | `Fade-in` & `Slide-up` | 从深黑渐现，带有轻微的纵向位移，体现“意识苏醒”。 |
| **状态/主题切换** | `CSS Transition (0.5s)` | 所有面板色彩平滑过渡，无生硬闪烁。 |
| **数据流转/MCP调用** | `Stroke Dashoffset` | 拓扑连线上光点闪烁流转，象征神经信号传递。 |

### 7.2 情绪驱动的动态环境 (Emotion-Driven Atmosphere)
UI 表现层将订阅 Java 后端的 `State` 状态模型：
* **平静 (Normal)**: 呼吸灯频率 0.8s，主色调保持纯净的灵枢青，节点平缓漂浮。
* **高负载/疲惫 (Fatigue)**: 显存濒临极限或处理超长上下文时，全局呼吸灯放缓至 1.5s，主色调明度降低 15%，UI 呈现“低电量”的克制感。
* **警觉 (Alert)**: MCP 监测到本地致命异常（如项目构建失败、数据库宕机），状态栏边缘出现红/黄色脉冲发光效果，强打断通知用户干预。

## 8. 技术栈支撑 (Technical Stack)

为实现以上极致性能与视觉效果，前端选型如下：
* **核心框架**: `Vue 3` (Composition API, `<script setup>`) + `Vite` (极速 HMR)。
* **样式引擎**: `Tailwind CSS` (负责全局 Design Tokens 与原子化响应式布局)。
* **组件库**: `Naive UI` (高度可定制，自带极佳的 TypeScript 支持与深色模式切换能力)。
* **可视化引擎**: `Apache ECharts` (用于监控大盘) / `D3.js` 或 `Vis-network` (专门攻坚复杂的力导向记忆图谱)。
* **状态与通信**: `Pinia` (本地偏好持久化) + `SSE / WebSocket` (与 Spring Boot 维持长连接，接收流式输出与日志推送)。
