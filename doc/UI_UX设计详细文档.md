# 灵枢 (Project LingShu-AI) 前端 UI/UX 详细设计文档

## 1. 概述 (Overview)

### 1.1 项目愿景
【灵枢】旨在从 0 到 1 打造一个具备长期记忆、情感演化与现实干预能力的本地化陪伴/协作智能体。前端界面作为用户与"数字生命"交互的唯一物理窗口，必须完美承载"生命感"（陪伴与呼吸）与"专业感"（全栈监控与极客控制）的双重职责。

### 1.2 设计目标
* **极致视觉 (Premium Aesthetics)**：打破传统管理后台 UI 的刻板印象，追求赛博朋克与东方禅意结合的视觉冲击力。
* **白盒化 (Observability)**：将复杂的 AI 思考链路、大模型推理耗时与记忆生长过程，通过可视化手段毫无保留地展示。
* **沉浸共生 (Immersion & Co-existence)**：通过动效、环境色渲染与 HUD (抬头显示) 元素，让用户感知到一个"活在本地"的数字灵魂。

## 2. 视觉风格与调性 (Aesthetics & Style)

### 2.1 风格定义：Cyber-Zen (赛博禅)
* **赛博 (Cyber)**：硬核的底层算力体现。包含 HUD 数据面板、发光边框、终端代码块、扫描线与点阵背景。
* **禅意 (Zen)**：中式名字的克制与呼吸感。包含空灵的背板色、磨砂玻璃 (Glassmorphism) 的通透感、流畅的贝塞尔曲线与有节奏的呼吸光效。

### 2.2 核心视觉元素
* **玻璃拟态 (Glassmorphism)**：使用高模糊度 (`backdrop-blur-xl`) 的毛玻璃作为各模块的悬浮背景板。✅ 已实现
* **动效氛围 (Atmosphere)**：包含网格背景的极缓慢平移、环境光偏移 (Mesh Gradient)、关键动作时的扫描线叠加。✅ Mesh背景已实现
* **HUD 装饰边界**：在屏幕角落、容器边缘添加伪代码（如 `SYS.MEM.ALLOC_`）、系统坐标、实时运行参数等装饰性等宽文本，营造"控制舱"沉浸感。⏳ 待实现

## 5. 核心交互组件 (Core Components)

### 5.1 记忆拓扑网络 (Graph-Explorer) ✅ 已实现

**文件位置**: `frontend/src/views/InsightView.vue`

* **节点设计**: 采用空心圆环、虚线描边与中心发光呼吸灯效果
* **边缘连线**: 使用带有张力的贝塞尔曲线，支持虚线动画
* **聚焦交互**: 鼠标悬停节点放大，触发光晕增强；点击后右侧栏同步解析记忆数据
* **右键菜单**: 支持查看关联详情、移除节点操作
* **详情面板**: 显示节点类型、关联置信度、活跃度、时间戳等信息

### 5.2 思考流可视化 (Chain of Thought Trace) ⏳ 待实现

* **形态**: 对话气泡上方弹出的微型折叠面板，类似终端命令行
* **作用**: 在大模型响应前，实时打印后端动作，打破黑盒
* **示例表现**: `[sys] 正在唤醒海马体... -> [neo4j] 匹配到历史节点 -> [llm] 开始生成回复`

### 5.3 全局极客指令盘 (Command Palette) ⏳ 待实现

* **形态**: 屏幕中央弹出的 Spotlight 风格磨砂黑框
* **交互**: 支持自然语言或以 `/` 开头的极客指令（如 `/mcp restart nginx`，`/mem flush today`）
* **触发**: `Ctrl + K` (或 `Cmd + K`) 随时唤醒
* **意义**: 赋予资深开发者最硬核、最快捷的本地系统控制权

### 5.4 认知权重刻度尺 (Cognitive Weight Ruler) ✅ 部分实现

**文件位置**: `frontend/src/views/InsightView.vue` (详情面板中的进度条)

* **形式**: 断点式的分段刻度进度条
* **意图**: 量化展示灵枢对当前用户情绪或某项知识点的"确信度"与"掌握深度"

### 5.5 对话界面 (Chat Interface) ✅ 已实现

**文件位置**: `frontend/src/components/chat/`

* **欢迎界面**: 灵魂球体动画 + 快捷操作按钮
* **消息气泡**: 区分用户/AI，支持 Markdown 渲染
* **输入框**: 底部浮动，支持发送消息
* **打字指示器**: 三点跳动动画

### 5.6 主题切换器 (Theme Switcher) ✅ 已实现

**文件位置**: `frontend/src/components/common/ThemeSwitcher.vue`

* 支持三套主题一键切换
* 主题配置持久化到 localStorage
* CSS 变量实时更新

## 7. 动效与情绪设计 (Animation & Emotion-Driven UI)

### 7.1 基础系统动效 ✅ 已实现

| 场景 | 动画方案 | 效果定义 | 状态 |
| :--- | :--- | :--- | :--- |
| **系统唤醒/页面入场** | `Fade-in` & `Slide-up` | 从深黑渐现，带有轻微的纵向位移 | ✅ |
| **状态/主题切换** | `CSS Transition (0.5s)` | 所有面板色彩平滑过渡 | ✅ |
| **呼吸灯效果** | `Keyframe Animation` | 灵魂球体、脉冲点等呼吸动画 | ✅ |
| **Mesh背景流动** | `Float Animation (45-60s)` | 背景色块缓慢移动 | ✅ |
| **消息入场** | `Slide-up (0.4s)` | 新消息从下方滑入 | ✅ |
| **节点交互** | `Scale & Glow` | 悬停放大、发光增强 | ✅ |

### 7.2 情绪驱动的动态环境 (Emotion-Driven Atmosphere) ⏳ 待实现

UI 表现层将订阅 Java 后端的 `State` 状态模型：
* **平静**: 呼吸灯频率 0.8s，主色调保持纯净，节点平缓漂浮
* **高负载/疲惫**: 显存濒临极限时，全局呼吸灯放缓至 1.5s，主色调明度降低 15%
* **警觉**: MCP 监测到本地致命异常，状态栏边缘出现红/黄色脉冲发光效果
