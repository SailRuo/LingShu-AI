# 贡献指南 (Contributing Guide)

感谢你对 **灵枢 (LingShu-AI)** 项目感兴趣！我们欢迎各种形式的贡献，包括但不限于：

- 提交 Bug 报告
- 提出新功能建议
- 改进文档
- 提交代码修复或新功能实现

## 1. 行为准则

请确保在参与社区讨论和贡献时保持礼貌和尊重。

## 2. 如何贡献

### 2.1 报告 Bug

如果你发现了 Bug，请在 GitHub Issues 中提交，并包含以下信息：
- 你的操作系统和环境版本（Java, Node.js, Docker 等）
- 复现步骤
- 预期行为与实际行为
- 相关日志或截图

### 2.2 提交功能建议

如果你有好的想法，欢迎提交 Issue 进行讨论。请描述：
- 该功能解决的问题
- 建议的实现方式（可选）

### 2.3 提交代码 (Pull Request)

1. **Fork** 本仓库。
2. **创建特性分支**：`git checkout -b feature/your-feature-name`。
3. **编写代码**：请遵循项目的代码风格和规范。
4. **运行测试**：确保所有测试通过。
5. **提交更改**：`git commit -m 'Add some amazing feature'`。
6. **推送到分支**：`git push origin feature/your-feature-name`。
7. **创建 Pull Request**。

## 3. 开发规范

### 后端 (Java)
- 使用 Java 21。
- 遵循 Google Java Style Guide。
- 确保代码有必要的单元测试。
- 使用 Lombok 简化代码。

### 前端 (Vue 3)
- 使用 TypeScript。
- 遵循 Vue 3 组合式 API (Composition API) 规范。
- 使用 Tailwind CSS 进行样式开发。

## 4. 许可证

通过提交 Pull Request，你同意你的贡献将遵循项目的 [GPLv3 许可证](LICENSE)。
